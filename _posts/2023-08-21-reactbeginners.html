React, udviklet af Facebook, er et populært JavaScript-bibliotek til at skabe interaktive brugergrænseflader ved hjælp af genanvendelige komponenter.

Med vanilla javascript kan det nemt blive uoverskueligt at arbejde med Document Object Model (DOM) - derfor bruger man et bibliotek som react, hvor man bruger komponenter, istedet for at manuelt opdatere DOM’en. Komponenter hjælper os med at skrive genanvendelige, modulære og bedre organiseret kode. Hver sektion på en hjemmeside kan blive til et komponent, fx menu eller et grid med flere komponenter inden i. React fokuserer kun på at bygge brugergrænseflader, derfor kræver det ofte yderligere biblioteker til funktioner som routing, HTTP-kald eller animationer.

Der er to tools man kan bruge til at lave en react applikation: create react app (CRA) og vite. I denne tutorial bruger man vite. Man kan installere den i command promt. I denne tutorial bruges TypeScript i stedet for JavaScript. TypeScript hjælper med at fange fejl tidligere i udviklingsprocessen, da TypeScript oversættes til JavaScript ved kompilering. Derudover specifcerer man typerne i typescipt, når man angiver typer i TypeScript, f.eks. for events, funktioner, objekter osv., giver det et ekstra lag af sikkerhed mod type-relaterede fejl. Dette betyder, at man kan fange potentielle fejl tidligt i udviklingsprocessen, hvilket kan spare tid og forhindre uforudsete fejl. Man kan bygge React-komponenter ved hjælp af TypeScript-filer, der har enten .ts eller .tsx filtyper., “.tsx" er den anbefalede filtype for React-komponenter, der bruger JSX sammen med typescript. JSX er en syntaks, der ligner XML, der bruges til at beskrive UI-komponenter i JavaScript, og derved opbygge brugergrænsefladen.

Komponenterne er ofte funktionsbaset, selvom man også kan bruge klasser. I React bruges JSX (JavaScript XML) til at beskrive, hvordan en komponents brugergrænseflade (UI) skal se ud. JSX kombinerer HTML og JavaScript på en måde, der gør det nemt at integrere dynamisk indhold i brugergrænsefladen.

Det blev forklaret, hvordan React dynamisk opdaterer indholdet i en applikation, hvor der både er en virtuel DOM og den faktiske DOM. Når du skriver JSX i en React-komponent, bliver det kompileret til javascript-kald, som skaber elementer i den virtuelle DOM. React bruger denne virtuelle DOM til at opdatere den faktiske DOM i browseren, fordi der bliver sammenlignet mellem den virtuelle og den faktiske, så hvis den virtuelle DOM ser anderledes ud, bliver den faktiske DOM opdateret.

En udfordring i React er, at en komponent ikke kan returnere mere end et element. For at returnere flere elementer skal de være indpakket i et enkelt overordnet element. I stedet for at indpakke flere elementer i en div, hvilket tilføjer et ekstra element til DOM'en, tilbyder React Fragments til at gruppere flere elementer uden at tilføje ekstra elementer til DOM'en. I stedet for et traditionelt loop benytter React map-metoden til at konvertere og gengive hvert element i et array. React kræver også, at hvert element har en unik key for at spore elementer og bestemme, hvilke dele af siden der skal opdateres. I react har hvert DOM element events som “onClick” til at håndtere events, man kan bruge en “arrow function”(ligner en lambda funktion) til dette.

I React findes der to hovedkoncepter: properties (ofte forkortet til "props") og state. Props fungerer som input eller argumenter, der gives videre til komponenter. State refererer til det indre data i en komponent og kan ændres. Når både props og state ændres bliver DOM’en opdateret. Når man skaber dynamiske komponenter, tillader React brugen af 'children' props. Dette gør det muligt for udviklere at videregive indhold inden i tags for komponenter, hvilket gør det mere læsbart og fleksibelt, især for længere indhold eller HTML.

For at håndtere dynamiske tilstande i komponenter tilbyder React en såkaldt “hook” som i praksis bliver brugt som fx. useState og useEffect. Denne hook gør det muligt at erklære state variables, som React kan overvåge for ændringer. Den retunerer et array, som har en konstant der peger på en variabel og en funktion til at opdatere denne variabel.
